IARM: Inter-Aspect Relation Modeling with Memory Networks in Aspect-Based Sentiment Analysis

EMNLP-2018 [paper]

Introduction

本文也是提出了融合临近的 aspects 的相关信息，用于ABSA任务。

如句子 “The menu is very limited - I think we counted 4 or 5 entries.” 中的子句 “I think … entries” 没有表达 “entries” 的真实情感，除非考虑到了另外一个 aspect “menu” 。这里的 “menu” 导致了 “entries” 具有相同的负面情感。我们假设我们的架构迭代地模拟来自其他 aspects 的影响，以生成准确的目标 aspect 表示。

另外，还考虑到了连词的影响 and, not only, also, but, however, though, etc.，句子中含有这些连词的话，aspects 往往会共享它们的情感，例如：“Food is usually very good, though I wonder about freshness of raw vegetables” 中的 “food” 是正向情感，而由于存在连词 “though”，“raw vegetables” 的情感为负向。

为了建模这些场景，首先，我们使用 GRU 和注意力机制独立地为所有的 aspect 生成 aspect-aware 的句子表示。然后，我们使用记忆网络重复地将目标aspect表示与其他aspect相匹配，以生成更精确的目标aspect表示。经过 refined 的表示送入softmax分类器用于最终的分类。

##Method

Problem Deﬁnition

Input

给定一个句子 S=[w1,w2,...,wL]S=[w_1,w_2,...,w_L]S=[w1​,w2​,...,wL​] ，wiw_iwi​ 是词，LLL 是句子中词的数量。同样，给定的 aspect-terms 为 A1,A2,...,AMA_1,A_2,...,A_MA1​,A2​,...,AM​ ，Ai=[wk,...,wk+m−1]A_i=[w_k,...,w_{k+m-1}]Ai​=[wk​,...,wk+m−1​] ，1≤k≤L,0<m≤L−k+11 \leq k \leq L, 0<m\leq L-k+11≤k≤L,0<m≤L−k+1 ，MMM 是句子中的 aspects 的数量。

Output

输出为每个 aspect-term 的情感极性 (1 for positive, 0 for negative, and 2 for neutral) 。

Model

我们的模型与其他文献的主要区别在于考虑句子中与目标 aspect 相邻的 aspects 。假设我们的 inter-aspect relation modeling (IARM) 架构建模了目标aspect和周围的aspects之间的关系，同时过滤了不相关的信息。

Overview

我们的IARM模型可以概括为以下步骤：

Input Representation

使用预训练的 Glove word embeddings，对于多词的 aspect-terms，我们使用它们的平均池化的结果作为 aspect 表示。

Aspect-Aware Sentence Representation

将 aspect 表示拼接到句子中的每个词，然后再将这个词序列送入 GRU 用于 上下文传播，接下来经过一个注意力层，以获得 aspect-aware 的句子表示。

Inter-Aspect Dependency Modeling

我们使用记忆网络去建模目标aspect和句子中其他的aspects的依赖。这是通过将 target-aspect-aware 的句子表示与其他 aspect 的 aspect-aware 的句子表示相匹配来实现的。经过一定数量的记忆网络迭代后，我们获得了与目标aspect的情感分类相关的句子的精确表示。进而将这个表示送入到softmax层进行分类。
