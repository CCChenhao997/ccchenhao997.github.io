Relational Graph Attention Network for Aspect-based Sentiment Analysis

Relational Graph Attention Network for Aspect-based Sentiment Analysis

ACL-2020 [paper]

School of Data and Computer Science, Sun Yat-sen University, China

Alibaba Group, China

Introduction

大多数最近的研究采用了基于注意的神经网络模型来隐式地将 aspects 与 opinion words 联系起来。然而，由于语言的复杂性和一句话中多个 aspects 的存在，这些模型往往混淆了它们之间的联系。

本文通过对句法信息的有效编码来解决这一问题。首先，我们通过重塑和修剪一个普通的依赖解析树，定义了一个统一的 aspect-oriented 的依赖树结构，该依赖树的根在目标方面上。然后，我们提出了一个关系图注意网络（R-GAT）来编码新的树结构用于情感预测。

之前也有一些研究工作使用了依赖树，但是他们都忽视了这一方法的缺点。

  • 首先，忽略了依赖关系，这种依赖关系可能表示 aspects 和 opinion words 之间的联系。
  • 其次，根据经验，只有解析树的一小部分与此任务相关，不需要对整个树进行编码。
  • 最后，编码过程依赖于树，使得优化过程中的批处理 (batch) 操作不方便。

本文对句法信息进行了重新审视，认为揭示与任务相关的句法结构是解决上述问题的关键。我们提出了一个新的 aspect-oriented 的依赖树结构，该结构分三个步骤构建。

  • 首先，我们使用普通解析器获得句子的依赖树。
  • 其次，我们重塑依赖树以将其 root 在 target aspect。
  • 最后，对树进行剪枝，只保留与 aspect 有直接依赖关系的边。

这种统一的的树结构不仅使我们能够关注 aspects 和潜在的意见词之间的联系，而且还方便了批处理 (batch) 和并行操作。

然后我们提出了一个关系图注意网络（RGAT）模型来编码新的依赖树。R-GAT 泛化了图注意力网络（GAT）以对带有标签的边的图进行编码。

The contributions of this work include:

  • We propose an aspect-oriented tree structure by reshaping and pruning ordinary dependency trees to focus on the target aspects.
  • We propose a new GAT model to encode the dependency relations and to establish the connections between aspects and opinion words.
  • The source code of this work is released for future research. [code]

Aspect-Oriented Dependency Tree

Aspect, Attention and Syntax

句子的句法结构可以通过依存关系解析来发现，这是生成依存树以表示语法结构的任务。单词之间的关系可以用有向边和标签来表示。我们使用三个示例来说明ABSA中 aspect，attention 和 syntax 之间的关系。

在第一个例子中，词 like 是一个动词，且它对于 aspect recipe 表达了正向的情感，这是基于注意力的 LSTM 模型成功的示例。然而，在第二个示例中将其用作介词时，该模型仍会以较高的权重对其进行处理，从而导致错误的预测。第三个示例显示了一个句子中的两个 aspects 具有不同的情感极性的情况。对于 aspect chicken ，LSTM 模型错误地将高的注意力权重分配给词 but 和 dried ，这将导致另一个预测错误。这些例子证明了基于注意力的模型的有限性。

通过在 aspects 和其他词之间引入显式的句法关系，可以避免此类错误。例如，如果模型注意到第三个示例中的 chicken 和 fine 之间的直接依赖关系，而不是 but ，则可能有所不同。

Aspect-Oriented Dependency Tree

上面的分析表明，与某个 aspect 具有直接联系的依赖关系可以帮助模型将更多的注意力集中在相关的意见词上，因此比其他关系更重要。同样，如上图所示，依赖树包含大量的语法信息，通常不植根于 target aspect 。然而，ABSA的重点是 target aspect ，而不是树的根。

基于上述观察，我们提出了一种新的 aspect-oriented 的依赖树结构，通过重塑原始依赖树，使其在目标方面生根，然后对树进行修剪，以丢弃不必要的关系。

对于一个输入的句子，我们首先使用一个依赖解析器去获得依赖树，rijr_{ij}rij​ 是节点 iii 到 节点 jjj 的依赖关系。然后，我们分三步构建一个 aspect-oriented 依赖树。

  • 首先，我们将 target aspect 放在 root 上，其中多词的 aspect 被视为实体。
  • 其次，我们将直接连接到 aspect 的节点设置为子节点，为其保留原始依赖关系。
  • 最后，其他依赖关系被丢弃，取而代之的是，从 aspect 到每个对应节点放置一个虚拟关系 n:con (nnn connected)，其中 nnn 表示两个节点之间的距离。如果距离超过4，则设置为 n=∞n=∞n=∞ 。如果句子包含多个 aspects ，我们将为每个 aspect 构造一个唯一的树。

这种 aspect-oriented 的结构至少有两个优点：

  • 第一，每个 aspect 都有自己的依赖树，并且可以减少不相关节点和关系的影响。
  • 第二，如果一个方面包含多个单词，依赖关系将在该方面聚合，这与需要额外的池化或注意力操作不同。

我们提出一个新的关系 n:con 的动机是，现有的解析器可能并不总是正确地解析句子，并且可能会错过与 target aspect 的重要连接。在这种情况下，关系 n:con 使新树更加健壮。
