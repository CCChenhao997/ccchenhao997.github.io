Syntax-Aware Opinion Role Labeling with Dependency Graph Convolutional Networks

ACL-2020 [paper]

Soochow University

ORL 任务可以看作是一个序列标注任务，旨在识别出意见持有者和意见目标。由于缺少标注的数据，本文中，我们试图通过比较和融合不同的表示来增强 ORL 模型的句法知识。我们也提出了 dependency graph convolutional networks (DEPGCN) 来对不同处理级别的解析器信息进行编码。 为了弥补解析器的解析结果不准确和减少错误传播，我们引入了多任务学习 (MTL) 来同时训练解析器和 ORL 模型。另外，我们发现句法知识的贡献并没有与BERT的表示完全重叠。

将句法依赖解析应用到NLP任务上存在两个阻碍：

1）解析结果的不准确；

2）pipeline 过程的错误传播。

为了克服第一个阻碍，我们使用的是所有依赖弧的概率矩阵（也可以视为 edge-weighted 有向图），而不是直接使用最后的离散输出（即单个最好的依赖树 single 1-best dependency trees ）。这种句法的概率表示提供更多的信息，减少了解析错误。

对于第二个障碍，考虑到 pipline 方法遭受错误传播问题，我们引入了多任务学习 (MTL) 框架。

总之，基于之前的研究使用句法来提升不同的任务，本工作研究了句法是否能够增强 ORL 模型。具体地，我们试图回答一下三个问题：

  • 如何有效地将不同的句法信息融合到 ORL 模型？
  • 如何减轻句法解析所带来的错误传播？
  • 句法知识是否已经被BERT的表示所覆盖了？

基于我们的实验，我们观察到：

1）与 single 1-best parse trees 相比，编码 edge-weighted graphs 可以实现更好的结果，因为模型能够对解析错误减少敏感，同时能够保持更丰富的结构信息；

2）融合不同的句法信息（显式的和隐式的）增强性能，并且MTL框架可以有效地减轻错误传播问题；

3）句法信息的贡献，特别是远距离依赖关系，并没有与BERT的表示完全重叠。

Basic Models

我们关注于根据 opinion expression 来识别 holders 和 targets ，利用 BiLSTM-CRF 作为我们的baseline。下图(a)展示了我们 ORL baseline 模型，由三个关键部分组成：input layer, the BiLSTM-based encoder, the CRF-based decoder 。

在编码层，我们使用一个三层的 BiLSTM 来编码句子，使用基于 CRF 的解码层输出全局最优序列标签。

The Biafﬁne parser

该解析器包含一个多层 BiLSTM 来编码输入的句子，接着是一个 biaffine transformation layer 用于计算所有 word pairs 的概率。然后搜索 highest-scoring 且通过 maximum spanning tree (MST) algorithm 形成树。

串联的三层：BiLSTM-based encoder, the biafﬁne scorer, the MST decoder 在不同的层级表达句法信息。编码器从输入的句子抽取出神经网络特征，输出隐藏状态 (HDN) ，这也可以被视为隐式信息。另一方面，1-best output parse tree 传递了显式句法结构。Biaffine scorer 为所有可能的依赖弧给出一个概率矩阵（也可以视为一个 edge-weighted 有向图），这比 1-best parse tree 具有更丰富的显式句法信息。

The Syntax-Aware Approach

本工作中，我们利用具有丰富句法结构的 edge-weighted graphs 来减轻解析错误的影响。具体地，我们首先利用 GCN 来编码 edge-weighted graphs ，然后将它们融合到具有隐式解析器隐藏状态的 ORL 模型的不同处理层级中。最后，我们进一步利用多任务 (MTL) 框架来减轻错误传播问题。

Dependency Graph Convolutional Networks (DEPGCN)

我们提出 dependency graph convolutional networks (DEPGCN) 更好地从 edge-weighted graphs 中编码句法信息。一方面，与显式的 1-best parse trees 对比，edge-weighted graphs 传递更丰富的结构信息 (by providing all latent syntactic structures) ，并且很好的避免了错误传播。另一方面，与解析器的隐式隐藏状态相比，即一个 edge-weighted graph，表示为一个注意力矩阵，显式地捕获 word pairs 的修饰强度。

如上图 (e) ，我们应用 DEPGCN 来连接解析器模型和 ORL 模型。我们首先从 biaffine parser 中的解码器中得到 edge-weighted graph ，然后将 graph 以临界矩阵 AAA 的形式送入到 DEPGCN 中。然后，我们将 ORL BiLSTM-based 编码器的输出作为初始输入 h0h^0h0 输入到 DEPGCN 中。最后，我们将 DEPGCN 的输出送入到 CRF-based 解码器中。

Combining Explicit and Implicit Syntax (DEPGCN+DEPHDN)

不同于显式的 1-best parse trees 或 edge-weighted graphs ，依赖项解析器的BiLSTM编码器的隐藏状态提供了有用的句法知识，并且对解析错误不那么敏感。而且使用隐式句法表示再下游任务中的有效性意见被证明了。在本节中，我们将描述如何将来自解析器隐藏状态的隐式句法信息和来自 edge-weighted graph 的显式句法信息融合到 ORL 模型中，以获得更好的性能。

我们首先简单的描述依赖解析器的隐藏状态（称为 DEPHDN）的使用。如 (d) 图，我们抽取解析器中编码器的输出，然后将他们送入 ORL 的 BiLSTM-based 编码器，作为额外的输入。解析器 BiLSTM 的每一层的隐藏状态作为句法表示，即 h1(l),...,hn(l)h_1^{(l)}, ..., h_n^{(l)}h1(l)​,...,hn(l)​ ，其中 hn(l)h_n^{(l)}hn(l)​ 是 BiLSTM 编码器的第 lll 层中词 wnw_nwn​ 的输出。然后，我们使用加权和操作得到单个向量 hisynh_i^{syn}hisyn​ 作为词 wiw_iwi​ 的最终句法表示。

Pipeline vs. Multi-Task Learning

图 (d-f) 三种方法可以以 pipline 形式或者 MTL 方式。特别地，pipline 方式首先训练依赖解析器，然后在训练 ORL 模型期间固定解析器组件。相反，以 MTL 方式同时训练解析器和 ORL 模型。除了 DEPGCN 方法，在本节中，我们探索了 MTL 方式来进一步减轻错误传播问题。

作为一个 baseline ，图© 是最常用的 MTL 方法，可以共享一个编码器，并使用多任务的输出层，称为 hard-parameter-sharing MTL 。然而，这种方法不适合我们的场景，在该场景中，辅助解析任务比主要 ORL 任务具有更多的标记数据，因为共享编码器极有可能偏向于解析性能。

我们采用图 (d-f) 的结构来保持任务模型参数独立。我们根据结合 ORL 和依赖解析的loss来更新模型的参数：

L=LORL+αLDEP\mathcal{L} = \mathcal{L}_{ORL} + \alpha \mathcal{L}_{DEP} L=LORL​+αLDEP​

α\alphaα 是一个语料权重因子，控制每个batch的依赖解析数据的loss贡献。

Experiment Setup

Dataset.

MPQA version 2.0 corpus

Dependency Parser.

使用 Stanford Parser v3.3.0 来标注 original phrase-structure Penn Treebank data ，然后将这个标注的数据来训练 biaffine parser 。另外，我们还用 Stanford POS tagger 来获得 POS tags 。

BERT.

将BERT最上一层的输出作为我们的额外输入。

Parameters.

100-dimensional glove embeddings

The BiLSTM layer number is set to 3, and the hidden output size is 200.

dropout=0.33

Adam

learning rate = 10−310^{-3}10−3

epochs=30, batch size = 50

对于MTL，我们依次训练 ORL 和句法分析的batchs，因为这种交织训练在我们的实验中可以获得更好的性能。此外，我们使用语料库加权技巧来平衡两个任务之间的数据量差距。
