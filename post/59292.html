关于hexo搭建后优化完善的一些问题

主页背景图片不能显示

        2019.8.19日上午，昨天搭建好的博客又意外显示不出主页的图片和站点其他图片。百思不得其所，明明昨天还好好地，今天怎么又出现问题了。

        正常显示应该是这样的:

        我在本地localhost打开博客是没有问题的，图片正常显示，说明问题是出在了github上。
(背景图用的是jon-bg.jpg)

        于是我去github上我存放图片的文件夹中看了一下，果然，缺少了主页显示的图片，只剩下一张20多kb的图。

        难道hexo deploy部署的时候比较大的图上传不上去吗？抱着这个疑问我测试了几次，只有一张27kb的图片部署成功了。

        我又google了一下，github上传文件限制是100M啊，为什么我十几兆的一张图片部署不上去呢？我怀疑问题应该不是出在图片大小的上面。

        我每次重新部署是用hexo clean && hexo g -d, 就是 clean genarater deploy 一步到位，我就一顿乱操作（还尝试过用 git add 来把图片上传上去，当然是不可能的😄），把指令拆开，没加hexo clean这一步，只写了 hexo g 生成静态文件, 然后 hexo d 部署到github上去, 就在这时候！！！奇迹发生了。。。

        本着以“测试工程师”的心态，又试了一遍 hexo clean && hexo g -d 命令，结果github上的图片又被删了。

        以上测试说明 hexo clean 指令可能会删除github上大的图片，或者说运行 hexo clean && hexo g -d 指令时，不能生成比较大的图片的静态文件，所以在部署的时候未上传到github上，才导致了我上述的问题发生。

总结：

        如果需要更新背景图片，我总结了一套指令方案:

  1. hexo clean 会把原本部署在github上较大的图片给删除掉
  2. 删除掉之后，我想重新上传图片的话，就需要按照以下操作:
    • hexo s 先运行本地服务，打开网页之后，再关闭服务
    • hexo g 运行该命令生成静态文件，千万不要加上clean
    • hexo d 将本地上的文件（github上没有的）部署到github上
    • hexo s 再次运行本地服务
    • 现在可以打开github pages的网址（或者直接查看github博客仓库里面的文件），你会发现ok了

博客中插入图片未能显示

        写这篇博客插入图片的时候遇到了这样的问题，所以也顺带记录下来。先是看了一篇插入博客图片的教程:

  1. 把主页配置文件_config.yml里的post_asset_folder: 这个选项设置为true;

  2. 在你hexo目录下执行这样一句话npm install hexo-asset-image --save，这是下载安装一个可以上传本地图片的插件;

  3. 等待一小段时间后，再运行hexo n "xxxx"来生成md博文时，/source/_posts文件夹内除了xxxx.md文件还有一个同名的文件夹;

  4. 最后在xxxx.md中想引入图片时，先把图片复制到xxxx这个文件夹中，然后只需要在xxxx.md中按照markdown的格式引入图片：
    ![你想输入的替代文字](xxxx/图片名.jpg)

        按照上面的步骤操作到了第四步，使用markdown命令插入图片后，打开博客网页发现图片没有显示出来，就是这样:

文件名称改动引发的博客页面404

        写完第二部分后，我手贱的将本文档的文件名称和存储图片的同名文件夹名称改了首字母，原本是 hexo完善及维护血泪史 ， 由于本人有强迫症，就改成了 Hexo完善及维护血泪史，然后hexo g, hexo d, 打开github pages，发现本博文一点开就是404 not found，在本地的服务打开网页又没有问题。难受呀，我是bug制造机吗？

        经过 苦思冥想，茶饭不思，昼夜难眠 之后，洗完澡后，打开电脑，打开github，查看那个博客的文件夹，果不其然！

       github上这个文件及图片文件夹名称没有更新，其原因是：在本地上文件的内容本身是没有改变的，只是改变了名字，所以在部署到github上时，该文件并没有更新，然后本地的文件确改变了名称，这导致了“线上”和“线下”的不匹配，然后打开github pages就是404了。
